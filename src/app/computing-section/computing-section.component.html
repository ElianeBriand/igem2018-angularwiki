Software : Plasmid Sketcher

Plasmid Sketcher at a glimpse

Plasmid Sketcher is a web-based software to draw plasmid map. In contrast to existing tools, the view is not generated from sequence information, but entirely from user input: this allows for non-proportional, didactic plasmid map that showcase function over technical precision.
Recommended use: to draw clear plasmid map for communication purpose (proposing a plasmid design, illustrating a poster, explaining a cloning process …). Use other software for technical operations.
You can use the software at open_in_new Plasmid Sketcher website
It is a Free and Open Source Software (FOSS) licenced under GPL v3 or later, with source code available here. It is based on Vixis’ angularplasmid and Google Angular Material.

Inspiration for Plasmid Sketcher

Visual communication has long been recognised as an integral part of biology, especially in bacteriology, cell biology and related disciplines. From the earliest papers, the textual description of restriction site positions, cloning protocols or operon structures have given way to the easier-to-grasp graphics we know and love.

The first plasmid maps

The concept of plasmid as transferable genetic material with circular topology emerged slowly. Mapping of plasmid took off with the discovery of type II restriction enzyme, starting with “endonuclease R” (now known as a mix of HindII and HindIII) in 1970 . The predictable and sequence-dependent cutting allowed characterization of fragment and later restriction cloning. Exploring the publication presenting the earliest widely-used plasmids like ColE1, pSC101 or pBR322, we see plasmids maps that look strikingly modern: except for dated typesetting, one could draw something similar for one’s publication.
Modern consumer-oriented computers were just starting to appear: these are not the work of a cloning assistant software like we have today, but more likely custom-made illustration. This is something we wanted to emulate, to have the possibility to deviate from one-size-fit-all representations to more streamline figures.

Visual Language Standardization

Some attempts have been made to standardize the visual language of synthetic biology , or protein domain description . Other proposals would use such language as a base for software modelling of biological processes . With the notable exception of pathway modelling, where the existence of exhaustive and widely-used database like KEGG and associated tools like CellDesigner have lead to de facto standardisation, other fields have not widely embraced such standardisation attempts.
We hypothesise this is the case because such proposals are not derived from a discipline-wide consultation: the normative power of these visual languages fails to manifest simply though lack of adoption.
Seen as simply descriptive reviews of visual elements in use, such publication are still worthwhile, and we used as inspiration for some elements of our software. We recognize the advantages of having only one type of visual representation for each feature, but given the absence of consensus, our software provides a variety of visual style when possible.

Putting the map in perspective

When analysing implicit goals rather than explicit visual choices, we find that there is a duality of purpose in plasmid maps:
  • Technical precision: that features are marked exactly, that displayed size are proportional to their length.
  • Clear Communication: that the overall purpose and functioning of the system is readily understood.
These purposes are sometime at odds. Smart compromise may be possible in some cases but not all. In synthetic biology, a high-level view is often valuable. Thus we wanted to offer a usable solution fulfilling the second identified goal.

About the software

Following the recognized user persona method , the interface was designed assuming a target audience of scientists or students, having some familiarity with the underlying biological concepts.

Technical considerations

The technology used is AngularJS, though there is a will to port it to Angular (“Angular 2.0”). It is based on Angular Plasmid by Vixis for plasmid visual rendering; and Google Angular Material for UI elements. It is an open source software licenced under the GNU GPL v3 or later. Source code available here.

Future features

While useable as of now, we plan to add further features, which will be published at the same URL. Future features :
